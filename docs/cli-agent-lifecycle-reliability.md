# CLI-Agent Sandbox Lifecycle Reliability (Omnigent comparison + adopted hardening)

> **Status:** ✅ IMPLEMENTED (pending merge) — 2026-07-06. Comparison verdict final; workstreams W1–W4 built + test-verified in one pass (BFF `failed` status + `status_errored` ingest, `last_event_at`, cli-agent-py `StopFailure→turn.failed` edge + reaper TOCTOU guard, Dapr-evidence-first session liveness reconciler). Whole-tree verification at completion: `pnpm check` 0 errors, TS vitest 575 files / 2697 tests pass, cli-agent-py pytest 255 pass / 0 fail. This doc records the **verified** comparison against Omnigent, the **concept→Dapr-native mapping** every change must follow, and the adopted / deferred / rejected decisions.
> **Scope:** lifecycle *observation* reliability for interactive-cli sandbox sessions (`claude-code-cli` family, `services/cli-agent-py`): turn-end/turn-failure edges, stall/crash detection, DB↔Dapr convergence, waiting/blocked semantics. Termination *authority* stays with the Lifecycle Controller (`docs/workflow-lifecycle-termination.md`, the lifecycle SSOT — this doc extends its observation half, it does not fork it).

---

## TL;DR

We audited **Omnigent** (github.com/omnigent-ai/omnigent @ `25307a9b`, read from a source clone — all claims below verified against real code, not the README) because it runs the same problem shape we do: real vendor CLIs (Claude Code, Codex, +18 others) in terminal panes, supervised for turn lifecycle, injection, and reaping.

**Verdict: our system is NOT wholesale less reliable — the two halves split cleanly.**

- **Where we are equal or stronger:** authoritative turn-end (both systems independently converged on *Stop-hook-exclusive* turn-end with pane/transcript quiescence demoted to a UI heuristic — ours at `services/cli-agent-py/src/hooks_api.py` `_process_locked`, theirs at `claude_native_forwarder.py` `_HOOK_EVENT_TO_STATUS`); and termination safety, where our durable, cross-app, fail-closed cooperative cascade + positive-evidence wedge force-finalize (`src/lib/server/lifecycle/{index,cascade}.ts`) is strictly stronger than their in-process in-flight guard.
- **Where Omnigent is clearly more reliable — the *observation* half:** a turn-FAILURE edge (`StopFailure`→failed) we simply lack; dual turn watchdogs (idle-reset-on-real-progress + absolute ceiling) where we only probe herdr liveness; crash→DB convergence (their orphan sweep vs our sessions rotting `running`/`idle` forever after an OOM-kill — our reconciler `maybeAutoResumeSession` was fully implemented and unit-tested but had **zero callers**); and a status vocabulary with first-class `failed` (our `session.status_errored` event had **no ingest handler** at all).
- **What we adopt (W1–W4):** the *concepts*, implemented **Dapr-natively** (mapping table below). What we defer: turn watchdog, background-task drain, structured pending-input, agy verified submit. What we reject: in-pod parent-death watchdog/orphan sweep (one session per pod — kubelet owns teardown), `waiting` as a DB status.

## Part 1 — Verified Omnigent mechanics (what they actually do)

All references are Omnigent paths at commit `25307a9b`.

1. **Authoritative turn-end/failure edges from Claude hooks.** `claude_native_forwarder.py:250-253`: `_HOOK_EVENT_TO_STATUS = {"Stop": "idle", "StopFailure": "failed"}` — documented as "the authoritative turn-end edges", explicitly contrasted with the PTY ~1s quiescence heuristic which once fired premature completions that "idempotently locked out the real one". Hooks are injected per-invocation via inline `--settings <JSON>` on the CLI launch (`claude_native_bridge.py:1049-1160`, `augment_claude_args:1304-1373`); every hook appends a JSONL line the forwarder tails. Subscribed events: SessionStart, Stop, StopFailure, UserPromptSubmit, TaskCreated/TaskCompleted, PostToolUse(TodoWrite/TaskUpdate), PreCompact, MessageDisplay, PermissionRequest, PreToolUse:AskUserQuestion. Deliberately **no SubagentStop**; subagent Stop/StopFailure records are filtered by `/subagents/` transcript path so a finishing subagent can't flip the parent (`:2596-2613`).
2. **Background-task-aware idle.** The Stop payload carries `background_tasks[]`; count = entries whose status ∉ `{completed, failed, stopped, killed}` — **unknown/absent status counts as live** so the indicator can never under-count (`claude_native_bridge.py:2144-2255`). `idle + count>0 → waiting` (`claude_native_forwarder.py:2721-2723`).
3. **Subagent completion = 5s transcript quiescence** (`_SUBAGENT_IDLE_QUIESCENCE_S = 5.0`, `:1539-1590`) — self-described Phase-A heuristic pending an authoritative hook.
4. **Verified injection** (`claude_native_bridge.inject_user_message:2403-2537`): prompt-glyph poll gate before typing (0.15s interval); delivery via file-backed `tmux load-buffer` + bracketed `paste-buffer` (beats the ~16KB argv cap; interior newlines stay data); then poll that the draft *appeared* in the composer (≤5s), send Enter, poll that it *left*, re-sending Enter every 1s up to 10s, error if never submitted.
5. **Process manager** (`runtime/harnesses/process_manager.py`): lazy spawn per conversation; crash detect via `returncode` poll + lazy respawn (`:685-697`); **in-flight tracking** is a single live `resp_<uuid>` per conversation, cleared on *every* terminal path by one writer (`:852-896`); idle reaper 30min/60s-cadence skipping in-flight entries (`:1127-1210`); startup **orphan sweep** keyed on an `AP_PID` sentinel + `os.kill(pid,0)` + `lsof` by socket, SIGTERM→3s→SIGKILL (`:1211-1307`).
6. **Parent-death is THREE layers** (`runtime/harnesses/_runner.py`): Linux `prctl(PR_SET_PDEATHSIG, SIGKILL)`; cross-platform ppid-poll watchdog (1s) that SIGTERMs itself; `os._exit` hard-exit timer if graceful shutdown wedges (`:91-112, 220-274`).
7. **Pane reaper with select→reconfirm** (`terminals/pane_reaper.py`): idle 30min; busy = active turn ∨ vendor-CLI autonomous run ∨ **human attached via tmux client list**; re-checks busy immediately before teardown (TOCTOU guard, `:228-233`); reap is pane-scoped, conversation resumes via `--resume`.
8. **Dual turn watchdog** (`runtime/harnesses/_scaffold.py:100-116,1435-1540`): idle timeout 240s reset by every non-heartbeat emit; absolute ceiling 3600s; **heartbeats explicitly do NOT reset the idle clock** (a wedged turn's heartbeats can't keep it alive); post-failures recorded so the watchdog attributes root cause.
9. **Status vocabulary + elicitations** (`server/schemas.py:1484-1491, 2292-2303, 1623-1640`): `{idle, running, waiting, failed}` where `waiting` = parked on async/background work, explicitly NOT human input; human approvals are a separate replayable `pending_elicitations`/`pending_inputs` index (created at the single publish chokepoint, resolved by answer/timeout, re-hydrated on cold load; per-elicitation GET route). Policy ASK gates park up to 24h; TOOL_CALL expiry fails **closed**.
10. **Codex turn-end is protocol, not scraping**: app-server JSON-RPC `turn/completed`→idle, `turn/failed`→failed with multi-signal failure override (error object / status field / empty turn) + stale-event rejection (`codex_native_forwarder.py:3529-3591`). Forwarder crashes are supervised with exp backoff + durable byte-offset/hash-fingerprint cursors for idempotent replay.

## Part 2 — Verified state of OUR system (pre-hardening baseline, 2026-07-06)

Ours paths: workflow-builder `main`; runtime = `services/cli-agent-py/src`.

**Strong (keep):**
- **Stop-hook-exclusive turn-end**, all 3 CLIs: claude via baked `docker/managed-settings.json` Stop hook → `:8002/internal/hooks/claude`; codex/agy via per-session relay `hooks.json` (`cli_adapters/base.py::write_hook_relay_script`). `hooks_api.py::_process_locked` drains the transcript tailer to quiescence, resolves last assistant text, raises `turn.completed` exactly-once per `(instance_id, turn)` onto the Dapr workflow. The transcript tailer is **content-only** by design (`transcript_tailer.py` — "Turn completion is owned EXCLUSIVELY by the Stop hook"); herdr `working|idle|blocked|done` is a 2s-debounced UI mirror, `done` coerced to `idle` (wfb #133), idle double-emit suppressed on the Stop path.
- **The subscription-leak fix** (`session_workflow.py` idle-probe loop): ONE `wait_for_external_event(LIFECYCLE_EVENT_NAME)` created before the loop, re-armed only after consuming a batch. Recreating per probe cycle leaked a pending subscription per cycle and durabletask delivers to the OLDEST subscription — the historical long-turn hang. **Structural invariant; nothing may add a second subscription.**
- **Fail-closed termination** (BFF Lifecycle Controller) + idle reaper with attach-awareness (`session_supervisor.py::_should_reap`: committed idle > `CLI_IDLE_TTL_SECONDS=3600` ∧ `attach_count==0` ∧ no recent terminal activity) + herdr-server/pw-mcp crash respawn with backoff + injection serialization behind one pane-write lock + codex/claude kickoff via deterministic paste→UserPromptSubmit-ack closed loop + premature-idle protection generalized to all 3 CLIs (`trust_idle_ready_fallback=False` + prompt-ready markers) + codex/agy credential capture watchers.

**Weak (the gaps this work closes / records):**

| Gap | Evidence (pre-fix) |
|---|---|
| No turn-FAILURE edge — Stop always means "ended cleanly"; failure inferred only from `cli.exited{exitCode≠0}` or output-sync timeouts | `hooks_api.py` had no StopFailure branch; `managed-settings.json` subscribes Stop only |
| `session.status_errored` consumed nowhere — never flips the DB row | no branch in `workflow-data.ts::ingestSessionEvent`; only read by benchmark outcome derivation |
| No `failed` session status; `NON_GRACEFUL_TERMINAL_STATUSES=["failed","error","crashed"]` read by resume predicates but **never written** | `src/lib/types/sessions.ts:1-8`; `sessions/resume.ts:26` |
| Crash→DB convergence absent: OOM-killed / out-of-band-deleted pod leaves the row `running`/`idle` forever | `lifecycle/auto-resume.ts::maybeAutoResumeSession` fully implemented + tested, **zero callers**; `sandbox-gc` is kubectl-only (no DB writes); `stuck-workflow-watchdog` excludes `agent-host-*` pods |
| Mid-turn stall on a live pane undetected — only the 120s herdr-liveness probe (`cli_lifecycle.py::probe_cli_activity`) + 4h age GC | no progress ledger; no cli-agent-py equivalent of dapr-agent-py's `session_host_monitor.py` |
| Wedge force-finalize (`shouldForceFinalizeCrossAppWedge`) fires only off a Stop request — unattended wedges never swept | `lifecycle/index.ts` keyed on `stop_requested_at` |
| Background tasks untracked — `autoTerminateAfterEndTurn` terminates at Stop with background work still running | no `background_tasks` handling anywhere in cli-agent-py |
| Blocked/approval = boolean+reason on an idle event + toast; DB row just `idle`; no structured, replayable pending-input record | `session_supervisor.py::classify_blocked_reason`; ingest maps blocked idle → `status="idle"` |
| `session.turn_heartbeat` consumed only by benchmark timings; no `last_event_at`; no absence-of-events alarm | `benchmarks/timings.ts` |
| Idle reaper select→act TOCTOU window (no reconfirm under the pane lock) | `session_supervisor.py::request_graceful_exit` |
| agy submit unverified post-paste (claude/codex have hook-ack) | `_confirm_submitted` `"more lines"` marker only detects multi-line drafts |

## Part 3 — Concept → Dapr-native mapping (binding for all lifecycle work)

Omnigent is a single-host asyncio system; we are a Dapr-workflow-wrapped fleet. We adopt their *concepts* on **our** primitives:

| Omnigent concept | Their mechanism | Our Dapr-native implementation |
|---|---|---|
| StopFailure→failed edge | hooks.jsonl tail → HTTP post | Hook relay → the **existing workflow external-event lane** (`raise_lifecycle_events` → the single re-armed `wait_for_external_event`); `session.status_errored` published via the existing **durable publish activity**, deterministic `source_event_id`, replay-safe |
| Crash detection / orphan sweep | PID sentinel + `os.kill(pid,0)` + lsof | **Dapr workflow runtime-status probe** through the Lifecycle Controller's existing per-session handles (`sessionHostAppId`, `{runtimeAppId, instanceId}`, `DURABLE_RUNTIME_MISSING_STATUS`, `TERMINAL_DURABLE_RUNTIME_STATUSES`) + K8s Sandbox CR/pod checks as corroboration + `last_event_at` silence |
| Safe convergence of dead sessions | SIGTERM→grace→SIGKILL | **The existing cascade** (`runDurableCascade`: cooperative → terminate fan-out → purge, purge-force when worker gone) with a `crashed` finalization variant — never a bespoke DB flip that leaves Dapr state divergent; single stop authority preserved |
| Periodic reconciler tick | in-process asyncio loop | **Dapr Jobs API + Scheduler** (per `docs/event-driven-workflow-triggers.md` — durable, replica-deduplicated; beats per-replica cron); K8s CronJob→internal endpoint kept as fallback idiom |
| Turn watchdog (deferred) | thread timers | Dapr **durable timers** in the existing idle-probe loop; stall snapshot read via the probe **activity** (the sanctioned nondeterminism boundary); heartbeats never count as progress |
| In-flight guard + select→reconfirm | resp-id map + busy recheck | Exactly-once completion keys (existing) + reconfirm-under-pane-lock in the idle reaper |

**Invariants:** no new `wait_for_external_event` subscriptions; deterministic `source_event_id`s on every publish; `is_replaying` guards on UI-only notifications; env knobs read at module import; anything marking a session terminal routes through the Lifecycle Controller.

## Part 4 — Decisions

**ADOPTED (W1–W4, landing now):**
- **W1** BFF: `failed` in `SessionStatus` (+`crashed` AND `error` stop-reason types — `error` is load-bearing: an interactive turn-failure idle must NOT coerce to `end_turn`, or the goal loop's `driveContinuationIfIdle` would auto-continue a failing turn; with `error` preserved the goal parks until the user intervenes); `session.status_errored` ingest handler (non-sticky — a live pod may recover to `running`; only `terminated` is sticky); consumer audit across capacity/benchmarks/resume/UI (`status-tone.ts` already mapped `failed`→danger; SSE terminal-statuses deliberately exclude `failed` so an interactive failed session's stream stays open).
- **W2** `sessions.last_event_at` (throttled 5s bump on ingest; backfill non-completed rows) — feeds the reconciler + truthful Observe "last active".
- **W3** cli-agent-py: `StopFailure` subscription (per-session settings.json — pinned Claude Code 2.1.201 verified to ship both `StopFailure` and `background_tasks`; **no image rebuild needed**, managed-settings bake is a follow-up) → `turn.failed` on the existing event lane, exactly-once shared completion key with Stop, `/subagents/` transcript-path filter on both edges; one-shot runs fail properly (same parent-visible path as `cli.exited≠0`), interactive sessions surface errored + idle and continue. Plus the idle-reaper select→reconfirm TOCTOU guard.
- **W4** Session liveness reconciler: Dapr-evidence-first (runtime status via per-session app-id), K8s CR/pod corroboration, tri-state evidence where **any `unknown` ⇒ skip** (positive-evidence rule inherited from `shouldForceFinalizeCrossAppWedge`); `converge_crashed` runs the full controller cascade (`lifecycle/index.ts::convergeCrashedSession` — purge mode + force-state-purge, CR reap, then `finalizeDb(reason, outcome:"crashed")` via the new `FinalizeOutcome` on the resolver closure → row lands `failed`+`stopReason:{type:"crashed"}`+`completedAt`); stop-intent rows route to `confirmDurableStop` (unattended-wedge sweep for free); Dapr-terminal-but-DB-nonterminal heals via `confirmDurableStop`'s all-closed finalize, no cascade; `maybeAutoResumeSession` finally gets its caller (gated off by default). **As-built layout:** pure core + runner `src/lib/server/lifecycle/session-reconciler.ts`; real deps/env wiring `application/adapters/session-reconciler-deps.ts` (re-exported through `application/session-reconciler-service.ts` — the `routes-no-adapters` boundary forbids routes importing adapters directly); ops/CronJob-fallback entry `POST /api/internal/sessions/reconcile`; Dapr Jobs callback `POST /job/session-liveness-reconcile` (always 200 — never trigger scheduler redelivery); job scheduled at BFF boot (`startup.ts`, best-effort `POST DAPR_HTTP_PORT /v1.0/jobs/session-liveness-reconcile`, `@every 10m`). Knobs: `SESSION_RECONCILER_{ENABLED(true), DRY_RUN(true), MIN_AGE_SECONDS(300), SILENT_WARN_SECONDS(900), AUTO_RESUME(off), MAX_ACTIONS_PER_RUN(10), TICK(dapr-job|cronjob|off)}`. Dry-run-first rollout.

**DEFERRED — evaluated 2026-07-06 (evidence-based verdicts):**
1. **Dual turn watchdog — DEFER; measure first.** The flagship long-run incident (claude GAN 163 min, `docs/gan-run-analysis-2026-06-30.md`) was a loop-EXIT parsing failure, not a mid-turn stall — it would not have been caught by a turn watchdog. One-shot runs are already bounded three ways (session self-timeout, Sandbox `shutdownTime`, and now the StopFailure edge + liveness reconciler); interactive stalls have a human attached. The reconciler's `SILENT_WARN_SECONDS=900` audit events are a free measurement instrument: **soak, count warn events on live pods, and build the watchdog only if real mid-turn stall incidence shows up.** Design if pursued: host-side progress ledger (hooks + transcript emissions + status changes; in-flight tool counting) via `probe_cli_activity`; stall `CLI_TURN_STALL_SECONDS≈600` / tool-relaxed `≈1800` / ceiling `3600`, warn-first, skipping `blocked`.
2. **Background-task drain — SPLIT: ship the count, defer the drain.** No evidence any one-shot persona spawns background tasks today; the JuiceFS mid-flight-write-loss risk is real but hypothetical. Cheap first step (~20 lines): count non-terminal `background_tasks[]` from the Stop payload (unknown counts as live) and ride `background_task_count` on `session.status_idle` — pure instrumentation that produces the data to justify (or permanently reject) the `CLI_BG_DRAIN_MAX_SECONDS` deferral machinery.
3. **Pending-input surfacing — PURSUE (highest-value deferred item).** Confirmed UX gap: a blocked session is invisible at list/fleet level (the detail-page toast is the only surface), and the new error-parking behavior (goal loop deliberately parks on `error` idles) increases the need to notice parked sessions from a list. Small, low-risk: `sessions.pending_input` jsonb cache maintained by the single ingest writer (set on blocked idle / tool-confirmation asks; cleared on running/answer/terminal) + list/fleet badge + session-scoped GET. Aligns with the Observe-hub direction (`docs/monitoring-ui-unification.md`).
4. **agy verified submit — DEFER indefinitely.** No documented dropped-message incidents since the `trust_idle_ready_fallback` fix; the existing marker set (`prompt_ready_marker`, `composer_draft_markers`, `idle_after_submit_is_success`) covers the empirically-discovered failure modes; agy is one of three runtimes. Revisit only on a reproduced incident.
5. ~~**managed-settings `StopFailure` bake**~~ — **SHIPPED in the same PR** (the outer loop rebuilds `cli-agent-py-sandbox` on that merge anyway; the bake closes the per-session-seed-failure hole; double registration is harmless via the shared completion key).

**REJECTED:**
- **In-pod parent-death watchdog + orphan sweep** — Omnigent needs it because many runner processes share one long-lived host. We run one CLI per sandbox pod; kubelet owns process-tree teardown; Sandbox `shutdownTime` + `workflow-builder-sandbox-gc` backstop pod leaks. An in-pod sweeper adds only false-kill risk. Revisit iff multi-session-per-pod ships.
- **`waiting` as a DB status** — Omnigent's `waiting` is machine-wait (background/async work), not human-wait. For us that's `idle` + `background_task_count` (data), and human-wait is pending-input (deferred item 3). A new status value would touch every live-status set for zero semantic gain.
- **CLI crash respawn in-pod** — a pane crash is terminal for the session by design (single-use pod); recovery is session-level Resume (`canResumeCliSession`) driven by the W4 reconciler + auto-resume policy, not process respawn.

## Part 5 — Rollout + verification

Order: W1+W2 (handler + column before events arrive) → W3 (normal cli-agent-py image release; flag `CLI_TURN_FAILED_EDGE_ENABLED` default-on; canary = one-shot smoke with an aborted turn → `session.status_errored` row + run fails, no hang) → W4 (`SESSION_RECONCILER_DRY_RUN=true` soak; then converge_crashed only; `AUTO_RESUME` stays off until separately validated).

E2E checks (dev cluster): kill a sandbox pod out-of-band → row converges `failed` + `stopReason=crashed`, Dapr instance purged, `session.reconciler_action` audit event on the stream, Resume appears; healthy sessions untouched across ticks; a kube/Dapr outage tick takes zero actions; the Dapr Job fires exactly once per interval across BFF restarts.

**Dev-rollout validation item:** the `/job/session-liveness-reconcile` callback authenticates via a token stamped into the Dapr Job `data` at scheduling time (constant-time compare against `INTERNAL_API_TOKEN`). This assumes daprd round-trips the job payload to the callback body — VERIFY on first dev rollout. Degrades safely if the assumption is wrong (callback 401s, no sweep runs; the internal-token `POST /api/internal/sessions/reconcile` + `SESSION_RECONCILER_TICK=cronjob` fallback remain available).
